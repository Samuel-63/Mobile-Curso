## Resumo do curso de Git e GitHub para iniciantes

Neste curso, aprendi os conceitos básicos de Git e GitHub, entendendo a diferença entre os dois e como eles ajudam no controle de versões de projetos.

O Git é uma ferramenta usada para acompanhar as alterações feitas nos arquivos de um projeto. Com ele, é possível salvar versões do código, voltar para alterações anteriores e organizar melhor o desenvolvimento.

Já o GitHub é uma plataforma online onde podemos armazenar repositórios Git, compartilhar projetos, trabalhar em equipe e manter o código disponível na nuvem.

Durante o curso, pratiquei comandos importantes como:

- `git init`, para iniciar um repositório;
- `git status`, para verificar o estado dos arquivos;
- `git add`, para preparar arquivos para o commit;
- `git commit`, para salvar alterações;
- `git push`, para enviar o projeto para o GitHub;
- `git pull`, para atualizar o projeto local com alterações do repositório remoto;
- `git clone`, para copiar um repositório existente.

Também aprendi sobre repositórios locais e remotos, criação de commits, envio de arquivos para o GitHub e a importância de escrever mensagens claras nos commits.
